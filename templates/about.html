Project Overview:

This project consists of three main components aimed at creating an end-to-end machine learning solution for detecting fraud in insurance data.

1. Exploratory Data Analysis (EDA) & Model Training:

The project begins with an Exploratory Data Analysis (EDA) of insurance data, sourced from Kaggle (https://www.kaggle.com/code/idolourie/car-insurance-eda/input). Using data engineering techniques, I extracted key features from the dataset and preprocessed the data for model training. A classification model based on XGBoost was then trained to predict the likelihood of fraud (fraud vs. not fraud). A prediction pipeline was built to handle raw data and generate predictions.
Technologies used: pandas, numpy, matplotlib, seaborn, plotly, sklearn, XGBoost.

2. Model Deployment as an API:

The trained XGBoost model was then containerized into a production-ready API using FastAPI. The model was deployed on Render.com, enabling it to serve real-time predictions.
Technologies used: FastAPI, Docker, Git, Pydantic.

3. Web Application - "The Tom Collection":

A web application called "The Tom Collection" was developed, featuring a form and a backend database to collect user data. The application allows users to send requests to the API and obtain predictions regarding insurance fraud qualification (fraud vs. not fraud).
Technologies used: Flask, SQLAlchemy, PostgreSQL, requests.
Frontend technologies: CSS, HTML.

All components of the project are published and available on GitHub for reference and further exploration.
